# lfguard 0.9.1

`lfguard 0.9.1` adds two advisory-only integration helpers for services and
agents that author or consume Lake Formation plans. It does not reintroduce
`lfguard apply` or any lfguard-owned AWS write execution path.

## Policy Authoring

- Adds `as_named_expression(...)` for filtered `LakePolicy` permission groups.
- Compiles a group filter into a reusable `lf_tag_expressions` definition plus
  database/table LF-Tag policy grants that reference the expression by
  `ExpressionName`.
- Preserves `catalog_id` and optional expression descriptions.
- Validates conflicting named expression definitions and rejects named
  expression groups whose filters cannot safely back both database and table
  grants.

## Request Evidence

- Adds `boto3_kwargs_for(change)` as a public, stateless marshaller for planned
  `Change` objects.
- Returns inert `{method, kwargs}` Lake Formation request evidence for
  consuming services that already own approval, credentials, retries, rollback,
  audit storage, and the actual AWS write decision.
- Keeps `AWSLakeFormationAdapter` read-only; no client is created and no AWS
  request is sent by the marshaller.

## Compatibility

- JSON plan/review contracts remain unchanged.
- The release is additive for CLI, library, and service integrations.
