# lfguard 0.9.0

`lfguard 0.9.0` makes the package advisory-only. The public product surface is
now review, audit, plan preview, explain, explain-batch, import, snapshot, and
read-only live inventory.

## Breaking Changes

- Removes the `lfguard apply` command.
- Removes saved-plan apply filtering options: `--only`, `--only-action`,
  `--max-changes`, `--max-destructive`, and `--execute`.
- Removes `ApplyResult` and `AWSLakeFormationAdapter.apply(...)`.
- Removes `additive-apply` and `destructive-apply` IAM templates from
  `lfguard permissions`.

## What Remains

- `lfguard plan` still emits planned changes, destructive markers, required
  planning flags, and `aws_api` evidence.
- `lfguard review` still writes a bundle with lint, audit, plan, planned grant
  evidence, summary, and manifest artifacts.
- `lfguard explain-batch` remains the access diagnosis contract.
- Live AWS support remains read-only for scoped inventory, snapshot, import,
  audit, plan, review, and explain workflows.

## Integration Boundary

Consuming services own approval workflow, AWS write IAM roles, grant/revoke
execution, and audit storage. `lfguard` provides evidence and recommended
actions; it does not grant or revoke Lake Formation permissions.
