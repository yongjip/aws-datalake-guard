# lfguard 0.7.0

`lfguard 0.7.0` shifts the package toward review evidence, exception
lifecycle, and operational access-decision contracts for service and LLM
integrations.

## Review Bundles

- Adds `lfguard review` to write one approval bundle containing `manifest.json`,
  `summary.md`, `summary.json`, `lint.json`, `audit.json`, `plan.json`, and
  `explain.json`.
- Records desired and current input provenance in `manifest.json`, including
  snapshot hashes and current-cache hashes when available.
- Uses `passed`, `review_required`, and `blocked` review statuses. Lint errors
  and destructive planned changes block; audit findings and safe planned changes
  require review.
- Defines `lfguard.review.explain.v1` as planned grant-change evidence. Use
  `lfguard explain-batch` for effective-access decisions.

## Batch Explain

- Adds `lfguard explain-batch --requests access-requests.json` for operational
  access questions across multiple principals and resources.
- Reports each request as `allowed` or `denied` with embedded
  `lfguard.explain.v1` evidence.

## Breaking Exception Metadata Change

- Policy exceptions now require `ticket`, `owner`, `approved_by`, `reason`, and
  `expires_at`.
- Expired exceptions remain lint errors. Exceptions expiring within 14 days and
  exceptions where `owner` equals `approved_by` are lint warnings.
