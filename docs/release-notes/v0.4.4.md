# lfguard 0.4.4

`lfguard 0.4.4` tightens named LF-Tag expression identity handling and keeps
lint and explain resolution behavior aligned.

## Fixed

- Duplicate named LF-Tag expressions with the same exact `(catalog_id, name)`
  identity are now rejected instead of being silently collapsed by shared state
  indexes.
- `lint` now emits `LF_TAG_EXPRESSION_DUPLICATE_IDENTITY` for duplicate
  desired-state LF-Tag expression identities.
- `explain()` now uses the same unscoped named-expression resolution rule as
  lint. An unscoped grant can resolve to a single unambiguous scoped named
  expression instead of incorrectly reporting the expression as missing.

## Tests

- Adds regression coverage for duplicate LF-Tag expression identity linting.
- Adds regression coverage proving `plan()` rejects duplicate exact expression
  identities.
- Adds regression coverage for `explain()` resolving unscoped grants to a
  single scoped named expression.

## Upgrade

```bash
python -m pip install --upgrade lfguard
lfguard --version
```
