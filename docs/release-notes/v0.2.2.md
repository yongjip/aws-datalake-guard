# lfguard 0.2.2

`lfguard 0.2.2` improves controlled production rollout workflows and release
trust signals.

## Added

- Adds stable plan JSON metadata:
  - `schema_version: "lfguard.plan.v1"`;
  - deterministic plan-local change IDs such as `change_001`;
  - risk, principal, resource, before/after, required flag, and boto3 API fields.
- Adds `lfguard apply --plan plan.json` for applying a reviewed saved plan.
- Adds selective apply with `--only change_001,change_004` or
  `--only-action grant.add_permissions`.
- Adds apply safety budgets with `--max-changes` and `--max-destructive`.

## Changed

- Saved destructive plan changes require the exact matching `--allow-*` flag
  before execution.
- Release automation now skips PyPI upload when the tagged version already
  exists and still verifies the package from PyPI.

## Upgrade

```bash
python -m pip install --upgrade lfguard
lfguard --version
```

For controlled apply:

```bash
lfguard plan --desired desired.json --output json --output-file plan.json
lfguard apply --plan plan.json --only-action grant.add_permissions --max-changes 10 --execute
```
