# lfguard 0.1.1

`lfguard 0.1.1` is a patch release that tightens Lake Formation permission
guardrails for LF-Tag table policies.

## Changed

- Blocks LF-Tag `TABLE` policy grants that combine `SELECT` with `ALTER`,
  `DELETE`, `DROP`, or `INSERT`.
- Blocks the same conflict when separate grants for the same principal and
  LF-Tag `TABLE` expression combine into `SELECT` plus table mutation
  permissions.
- Blocks mutation permissions on column-filtered `table_with_columns` grants.
- Documents the preventive pattern: read LF-Tag grants should stay
  `SELECT`/`DESCRIBE` only, while writer/admin permissions belong in separate
  controlled policies.

## Why It Matters

Lake Formation can treat a table-level LF-Tag read grant as partial-column
`SELECT` when one or more columns override the table's LF-Tag value. AWS does
not allow partial-column `SELECT` to be combined with table mutation
permissions such as `INSERT`, `DELETE`, `ALTER`, or `DROP`. This release blocks
that grant shape before apply.

## Upgrade

```bash
python -m pip install --upgrade lfguard
lfguard --version
```
