# lfguard 0.6.5

`lfguard 0.6.5` is a live AWS correctness release for real Lake Formation
permission inventories.

## TableWithColumns ColumnWildcard

- Adds state, schema, AWS conversion, and explain support for
  `TableWithColumns.ColumnWildcard` grants, including optional
  `excluded_columns`.
- Preserves all-column grants during live import, audit, plan, and apply instead
  of rejecting AWS responses that omit explicit `ColumnNames`.
- Uses table-level LF-Tag lookup for column-wildcard resources because Lake
  Formation rejects `GetResourceLFTags` requests containing
  `TableWithColumns.ColumnWildcard`.

## Live Inventory Robustness

- Skips non-taggable grant resources during resource-tag import discovery so
  catalog, data-location, and data cells filter grants do not produce invalid
  `GetResourceLFTags` requests.
- Sorts live resource-tag lookup resources by stable identity to avoid
  scoped/unscoped catalog comparison crashes.
- Falls back to catalog-level `ListPermissions` with client-side filtering when
  scoped principal/resource permission reads are denied.

## IAM Templates

- Adds `lakeformation:GetDataCellsFilter` and
  `lakeformation:ListDataCellsFilter` to the read-only template and preflight
  checks.
- Adds data cells filter create/update/delete actions to the additive and
  destructive apply templates.
