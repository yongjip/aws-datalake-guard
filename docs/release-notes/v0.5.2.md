# lfguard 0.5.2

`lfguard 0.5.2` hardens catalog-aware Lake Formation evidence and live AWS
request handling across LF-Tags, named LF-Tag expressions, resource tags, grants,
and data cells filters.

## Added

- Catalog-scoped LF-Tag definitions are planned, audited, linted, applied, and
  serialized independently, so same-name tag keys can exist in different
  catalogs without index collisions.
- `lfguard explain --data-cells-filter FILTER` can target Lake Formation data
  cells filter grants directly.
- Explain JSON includes catalog-specific effective LF-Tag evidence.
- Python policy bundles accept catalog IDs for tag keys, resource tags, LF-Tag
  policy bundles, catalog creator bundles, stewardship, admin, and data-location
  access intents.

## Fixed

- AWS current-state loading and import preserve the requested catalog ID when
  Lake Formation responses omit `CatalogId`.
- Resource tag apply requests include scoped LF-Tag pair catalog IDs.
- Resource-tag import discovers resources from Lake Formation grants even when
  grants are not included in the generated desired-state output.
- Explain uses grant, target, and effective catalog context for LF-Tag policy
  and named-expression evaluation instead of matching definitions across
  unrelated catalogs.
- Audit reports wholly unmanaged current LF-Tags and resource tag assignments
  with catalog-aware ownership and ignore behavior.
- Current broad `ALL`, `SUPER`, and `SUPER_USER` grants are treated as covering
  concrete desired permissions in audit, explain, and planning evidence while
  still being reported as unmanaged policy drift.
- Destructive broad-permission revokes first add desired concrete permissions so
  the resulting plan still converges to desired access.

## Tests

- Adds Stubber coverage for catalog-scoped LF-Tag inventory, LF-Tag apply,
  resource-tag apply, table grants, data-location grants, data cells filter
  grants, named LF-Tag expression grants, and catalog-preserving current-state
  load paths.
- Adds regression coverage for duplicate catalog-scoped identities, effective
  LF-Tag evidence by catalog, data cells filter explain behavior, broader
  current-grant coverage, and broad-permission convergence.

## Upgrade

```bash
python -m pip install --upgrade lfguard
lfguard --version
```
