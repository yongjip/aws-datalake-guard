# lfguard 0.3.0

`lfguard 0.3.0` expands the package from a strict diff/apply engine into a
more practical adoption tool for existing Lake Formation environments.

## Added

- Adds first-class named LF-Tag expressions in desired/current state with
  `lf_tag_expressions`.
- Adds LF-Tag policy grants that reference a named expression with
  `expression_name`.
- Adds audit drift detection for missing, changed, and unmanaged named LF-Tag
  expressions.
- Adds conservative planning and AWS adapter support for named LF-Tag expression
  create, update, and delete actions.
- Adds configurable desired-state lint severities through top-level `lint`
  overrides.
- Adds `ownership` and `ignore` config for unmanaged principals and resources.
- Adds `lfguard import` to scaffold starter desired state from live AWS
  LF-Tags, named LF-Tag expressions, grants, and grant-discovered resource tags.

## Changed

- Summary, check, validate, schema, state-format, AWS API, and IAM permission
  docs now include named LF-Tag expression support.
- Unmanaged current-state audit findings can now be scoped to managed
  principals/resources or suppressed for explicit legacy exceptions.
- Destructive named LF-Tag expression updates and deletes require exact
  `--allow-lf-tag-expression-updates` or
  `--allow-lf-tag-expression-deletes` flags.

## Upgrade

```bash
python -m pip install --upgrade lfguard
lfguard --version
```

Starter import for an existing account:

```bash
lfguard import \
  --catalog-id 123456789012 \
  --include lf-tags,lf-tag-expressions,resource-tags,grants \
  --output policy/imported-desired.json
```
