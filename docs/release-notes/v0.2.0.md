# lfguard 0.2.0

`lfguard 0.2.0` adds a Python-native policy authoring layer for teams that want
permission groups and tag assignments to be the source of truth, while still
generating normal reviewable desired-state JSON or YAML.

## Added

- Adds `lakeformation_guard.policy` with `LakePolicy`, `TagAssignmentScope`,
  user-defined permission groups, and IAM role bindings.
- Adds safe permission templates:
  - `reader()` for `DESCRIBE` plus `SELECT`.
  - `editor()` for whole-table read/write on existing tables.
  - `table_creator()` for table creation inside approved databases plus editor access.
  - `database_creator()` for catalog-level `CREATE_DATABASE` in narrowly scoped
    onboarding or catalog administration roles.
- Adds `tag_database()`, `tag_table()`, and `tag_columns()` helpers for explicit
  LF-Tag assignments in Python policy files.
- Adds mapping forms for LF-Tag keys that are not valid Python keyword
  arguments, such as `reader().where({"data-domain": "sales"})`.
- Adds `lfguard generate` to compile `policy.py` into JSON or YAML desired
  state.
- Adds `lfguard generate --check` for CI gates that keep `policy.py` and the
  generated desired-state file in sync.
- Adds optional `lf_tag_key_metadata` to generated desired state so lint can
  allow whole-table editor/table-creator grants while staying conservative for
  hand-written grants that may narrow columns.

## Changed

- Updates `lfguard bootstrap` to create `policy.py` as the source of truth and
  run `lfguard generate` before `lfguard check`.
- Updates examples and documentation around the smaller reader/editor/table-creator
  permission model and explicit database-creator naming.
- Removes the long `aws-lakeformation-guard` console alias. The installed CLI
  command is now only `lfguard`.

## Upgrade

```bash
python -m pip install --upgrade lfguard
lfguard --version
```

For policy-as-code repositories, generate the desired-state artifact and commit
both files:

```bash
lfguard generate policy.py --output-file policy/desired.json --force
lfguard check --desired policy/desired.json --fail-on-findings
```
