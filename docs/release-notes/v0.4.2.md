# lfguard 0.4.2

`lfguard 0.4.2` tightens catalog handling for named LF-Tag expressions and
catalog-scoped apply requests.

## Fixed

- Payload-level `catalog_id` now takes precedence over the adapter default
  catalog ID when building live boto3 apply requests.
- Named LF-Tag expression plan and audit indexes are now keyed by
  `(catalog_id, name)` instead of name only. Same-name expressions in different
  catalogs are no longer treated as equivalent.
- Live current-state loading now fetches named LF-Tag expressions by
  catalog-scoped keys from desired expressions and named-expression grants.
- `explain()` now resolves named LF-Tag expressions by catalog-scoped keys.

## Tests

- Adds botocore `Stubber` coverage for payload catalog IDs overriding adapter
  defaults.
- Adds planner, audit, live-loading, and explain regression tests for same-name
  LF-Tag expressions in different catalogs.

## Upgrade

```bash
python -m pip install --upgrade lfguard
lfguard --version
```
