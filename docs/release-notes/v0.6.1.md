# lfguard 0.6.1

`lfguard 0.6.1` is a customer-readiness patch for the 0.6.0 cache and data
cells filter release.

## Fixes

- Current-state cache hits now require both the desired-state fingerprint and a
  provider-context fingerprint to match. The CLI context includes the AWS
  provider, profile, region, and catalog ID, which prevents a cache generated
  for one environment from being reused for another.
- Cache envelopes now record `provider_context` and `provider_fingerprint`.
  Older 0.6.0 cache files safely miss and refresh because they do not contain
  the provider fingerprint.
- Safe `SELECT` grants on `data_cells_filter` resources no longer trigger the
  named database/table/column grant review lint rule. Mutating permissions on
  filtered grants are still blocked by the column-filter mutation guardrail.

## Docs and Samples

- `lfguard sample` now demonstrates a Lake Formation data cells filter
  definition, a filtered `SELECT` grant, an `explain --data-cells-filter`
  workflow, and scoped live cache usage.
- README and CLI docs now make data cells filters visible in the first-run path
  and recommend explicit `--profile`, `--region`, `--catalog-id`, and
  per-context cache paths for live cached workflows.
