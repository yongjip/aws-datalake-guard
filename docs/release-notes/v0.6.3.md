# lfguard 0.6.3

`lfguard 0.6.3` is a docs and examples release for customer adoption workflows.
It includes the governance operating-model material added after `0.6.2`.

## Docs

- Adds [`ci-evidence-workflows.md`](../ci-evidence-workflows.md), covering
  pull request gates, protected drift checks, plan review evidence, explain
  evidence, dry-run apply reports, and artifact handling.
- Adds [`terraform-cdk-coexistence.md`](../terraform-cdk-coexistence.md),
  clarifying how Terraform, CloudFormation, and CDK can own infrastructure
  while `lfguard` owns reviewed Lake Formation policy.
- Adds [`import-adoption-recipes.md`](../import-adoption-recipes.md), with
  practical recipes for evaluation imports, gradual ownership, Python policy
  conversion, data cells filter adoption, and environment promotion.
- Adds [`exception-lifecycle.md`](../exception-lifecycle.md), documenting
  request, review, evidence, apply, expiry, and removal patterns for scoped
  policy exceptions.
- Adds [`permission-request-bundles.md`](../permission-request-bundles.md),
  showing how repeated access requests can be represented as policy data
  without turning `lfguard` core into an approval workflow product.

## Examples

- Adds `examples/permission-requests.py`, a runnable Python policy example that
  models approved access requests and compiles them to ordinary desired state.
- Links the new operating-model docs from README, examples, adoption,
  positioning, framework, GitHub Actions, and roadmap pages.
