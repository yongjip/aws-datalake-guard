# lfguard 0.8.0

`lfguard 0.8.0` marks the advisory review and access-diagnosis contracts as the
primary service integration surface.

## Service And LLM Integration

- Adds `docs/service-integration.md` for DMS-style backend wrappers that invoke
  `sys.executable -m lakeformation_guard`, read JSON artifacts, enforce
  timeouts, and keep approval/application logic outside lfguard.
- Adds checked-in `review` and `explain-batch` contract fixtures under
  `examples/artifacts/` for adapter tests and LLM agent prompts.
- Reinforces the advisory contract: `severity` is a governance signal, while
  `recommended_action`, `hard_block`, and review `status` drive workflow
  decisions.

## Contract Positioning

- Keeps `review/explain.json` scoped to planned grant-change evidence.
- Keeps `explain-batch` as the operational access-diagnosis interface.
- Keeps `apply` explicit and conservative; no automatic apply or approval
  workflow behavior is introduced.

## Packaging

- Updates package metadata to beta maturity now that the service-facing review
  and explain contracts are documented and covered by fixtures.
