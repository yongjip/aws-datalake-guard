# lfguard 0.5.0

`lfguard 0.5.0` moves the package toward a stricter Lake Formation permission
framework with stable evidence outputs, explicit exception handling, and
generic permission bundles.

## Added

- Audit JSON now includes `schema_version: "lfguard.audit.v1"` and
  deterministic `finding_001` identifiers.
- Explain JSON findings now include deterministic `finding_001` identifiers.
- Desired state supports scoped policy exceptions with required reason, expiry,
  and owner or approver metadata.
- New policy bundle primitives: `producer()`, `steward()`, `admin()`, and
  `data_location_access()`.
- New permission framework documentation and examples for policy exceptions and
  generic bundles.

## Changed

- Mutating, grantable, and named-resource governance lint findings are errors
  by default unless covered by a matching non-expired exception or lint override.
- `validate` rejects duplicate named LF-Tag expression identities in desired
  state and current snapshots.

## Tests

- Adds regression coverage for stable audit and explain JSON fields.
- Adds coverage for policy exception parsing, validation, expiry handling, and
  lint suppression scope.
- Adds coverage for generic policy bundles and documentation examples.

## Upgrade

```bash
python -m pip install --upgrade lfguard
lfguard --version
```
