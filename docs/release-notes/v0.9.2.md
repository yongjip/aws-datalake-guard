# lfguard 0.9.2

`lfguard 0.9.2` fixes data cells filter request evidence so services and agents
do not receive boto3 kwargs that AWS would reject for missing `TableCatalogId`.
It remains advisory-only and does not add AWS write execution.

## Data Cells Filters

- Adds `DATA_CELLS_FILTER_MISSING_CATALOG_ID` as a hard-blocking lint finding.
- Flags desired `data_cells_filters[]` definitions without `catalog_id`.
- Flags grants on `data_cells_filter` resources without `resource.catalog_id`.
- Keeps model and schema parsing compatible; the requirement is surfaced through
  lint/review evidence rather than a parsing break.

## Request Evidence

- Makes `boto3_kwargs_for(change)` raise a clear `ValueError` when data cells
  filter create, update, delete, or grant evidence would omit AWS
  `TableCatalogId`.
- Does not infer `catalog_id` from STS or credentials. Consuming services must
  provide the intended catalog ID explicitly.

## Compatibility

- JSON review/plan schema versions remain unchanged.
- Existing valid data cells filter states with `catalog_id` are unaffected.
