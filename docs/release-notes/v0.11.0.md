# lfpolicy 0.11.0

`lfpolicy 0.11.0` expands the read-only AWS adapter and adds advisory migration
readiness evidence for Lake Formation IAM compatibility coverage. It does not
add AWS write execution or approval orchestration.

## Scoped data cells filter reads

- Adds `AWSLakeFormationAdapter.list_data_cells_filters(database_name,
  table_name, catalog_id=...)` for one explicitly named table.
- Uses paginator or `NextToken` fallback behavior and returns a deterministic
  tuple of modeled `DataCellsFilterDefinition` records.
- Gives an explicit catalog ID precedence over the adapter default and does not
  crawl databases or tables.

## IAM compatibility readiness

- Discovers `IAM_Allowed_Principals` coverage for explicitly desired databases
  and tables during live current-state inventory.
- Emits stable `IAM_ALLOWED_PRINCIPALS_PRESENT` audit evidence with
  `recommended_action: "review_required"` and `hard_block: false`.
- Reports compatibility coverage in access explanations as non-decisive
  `context`; AWS IAM authorization is not modeled, so the evidence alone does
  not produce an allowed decision.
- Excludes the compatibility principal from generated revoke plans. lfpolicy
  does not convert, revoke, or otherwise mutate Lake Formation permissions.

## Compatibility

- Existing JSON schema versions and required fields remain unchanged.
- The new audit finding and explain context are additive advisory evidence.
- The PyPI distribution, CLI command, and import package remain `lfpolicy`.
