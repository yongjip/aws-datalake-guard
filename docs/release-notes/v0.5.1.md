# lfguard 0.5.1

`lfguard 0.5.1` fixes catalog-scoped catalog grants in live AWS conversion and
keeps explain evidence scoped to compatible catalogs.

## Fixed

- `admin(catalog_id=...)` and other catalog-scoped catalog grants now preserve
  the catalog ID in live apply by sending both the request `CatalogId` and
  `Resource.Catalog.Id`.
- Live inventory now preserves `Resource.Catalog.Id` when converting Lake
  Formation catalog grant resources back into `ResourceRef`.
- Desired resource catalog IDs now override an adapter default catalog ID for
  resource tag and grant requests.
- `explain()` no longer reports a catalog-level grant as context for a target in
  a different catalog.

## Tests

- Adds botocore `Stubber` coverage for catalog-scoped grant apply and catalog
  grant inventory.
- Adds explain regression coverage for same-catalog context and
  different-catalog exclusion.
- Adds catalog resource conversion round-trip coverage.

## Upgrade

```bash
python -m pip install --upgrade lfguard
lfguard --version
```
