# lfguard 0.1.0

`lfguard` is an opinionated AWS Lake Formation guardrail package for teams that
want LF-Tag policy reviewed as code, checked for drift, planned before apply,
and changed conservatively.

## Install

```bash
python -m pip install lfguard
python -m pip install "lfguard[aws,yaml]"
```

## First Run

Try the offline demo without AWS credentials:

```bash
lfguard sample --output-dir lfguard-demo --include-ci
lfguard check \
  --desired lfguard-demo/desired.json \
  --current-snapshot lfguard-demo/current-snapshot.json
lfguard plan \
  --desired lfguard-demo/desired.json \
  --current-snapshot lfguard-demo/current-snapshot.json
```

Start a policy repository from a template:

```bash
lfguard bootstrap --output-dir lfguard-policy
lfguard init --output-file policy/desired.json
lfguard init --template blank --output-file policy/desired.json
```

## Highlights

- Audits desired LF-Tag policy against current state from files or live AWS.
- Lints desired policy for undefined LF-Tag keys and values before AWS access.
- Summarizes policy inventory for pull request and release review.
- Produces conservative plans that add missing LF-Tags, tag assignments, and
  permissions by default.
- Keeps destructive operations behind explicit allow flags.
- Supports JSON and YAML desired/current state files.
- Emits text, JSON, Markdown, and SARIF reports for CI, pull requests,
  dashboards, and artifacts.
- Includes GitHub Actions helpers such as `--github-summary` for check, lint,
  summary, audit, plan, and apply reports, plus `--fail-on-findings` and
  `--fail-on-changes` gates.
- Adds `lfguard check` for one-step offline validation and desired-policy lint
  gates.
- Uses `lfguard check` in generated policy/demo workflows so new users get the
  same offline gate locally and in CI.
- Emits bash, zsh, and fish completion scripts with `lfguard completion`.
- Generates starter IAM policies with `lfguard permissions` for read-only,
  additive apply, and destructive apply workflows.
- Bootstraps a policy repository layout with schema, CI, pre-commit, and rollout
  README files.
- Generates an optional offline GitHub Actions demo workflow with
  `lfguard sample --include-ci`.
- Ships copyable GitHub Actions workflows for drift checks and optional Code
  Scanning SARIF uploads.
- Provides a typed Python API under `lakeformation_guard`.
- Installs the short `lfguard` CLI plus the descriptive
  `aws-lakeformation-guard` alias.

## Safety Defaults

`apply` is a dry run unless `--execute` is passed. Permission revokes, LF-Tag
value removals, and resource tag removals are not planned unless the matching
allow flag is enabled.
