# lfguard 0.8.1

`lfguard 0.8.1` hardens the advisory evidence contract introduced in 0.8.0.
It remains focused on service, CI, ticket workflow, and LLM-agent consumption;
no automatic apply or approval workflow behavior is added.

## Advisory Evidence Contracts

- Adds `docs/finding-catalog.md` with stable lint finding, audit finding, and
  plan action metadata.
- Adds `title` and `docs_url` to lint, audit, plan, and planned grant-change
  JSON evidence.
- Adds `summary.json` evidence metadata for review bundles, including
  `generated_at`, `lfguard_version`, input source/hash metadata, and truncation
  status.
- Adds compact `diagnosis` fields to `explain-batch` results for matched
  findings, missing permissions, matched sources, desired-grant gaps, and notes.

## Fixtures And Schemas

- Adds JSON Schema documents for the service-facing report contracts under
  `docs/schemas/`.
- Adds review-case golden fixtures for `passed`, `review_required`,
  `approval_required`, and `blocked` outcomes.
- Adds explain-batch golden fixtures for allowed, missing-permission denied, and
  no-matching-grant denied outcomes.
- Adds tests that verify catalog coverage, schema compatibility, fixture
  contracts, and advisory decision semantics.

## Release Safety

- Updates the PyPI release workflow so pushing a tag alone does not publish.
- Publishing now requires a GitHub Release or explicit manual workflow dispatch.
