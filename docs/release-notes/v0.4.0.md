# lfguard 0.4.0

`lfguard 0.4.0` adds an integration boundary for current-state loading and a
new access explanation workflow for Lake Formation policy reviews.

## Added

- Adds `CurrentStateProvider`, a small protocol for tools that provide current
  Lake Formation state from files, APIs, caches, or databases.
- Adds `SnapshotCurrentStateProvider` and `SnapshotFileCurrentStateProvider`
  for offline and test workflows.
- Adds the importable `explain()` API for one principal/resource access
  question.
- Adds `lfguard explain` with text, JSON, and Markdown output.
- Explains direct grants, LF-Tag policy grants, named LF-Tag expression matches,
  effective LF-Tags, data-location grant context, requested permission gaps, and
  desired grants missing from current state.

## Changed

- CLI current-state loading now goes through the provider interface.
- Architecture and report-format docs now describe provider integrations and
  explain report payloads.
- CI and release smoke tests cover the new `explain` command and public API.

## Upgrade

```bash
python -m pip install --upgrade lfguard
lfguard --version
```

Example access explanation from a reviewed snapshot:

```bash
lfguard explain \
  --desired policy/desired.json \
  --current-snapshot snapshots/prod-current.json \
  --principal arn:aws:iam::111122223333:role/Analyst \
  --database analytics \
  --table orders \
  --permissions SELECT \
  --output markdown
```
