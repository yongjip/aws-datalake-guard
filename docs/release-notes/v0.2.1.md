# lfguard 0.2.1

`lfguard 0.2.1` strengthens adapter testing and release confidence. Runtime
behavior is unchanged.

## Added

- Adds botocore `Stubber` contract tests for Lake Formation adapter request
  shapes without contacting AWS.
- Adds optional Moto emulator tests for small adapter round trips.
- Adds opt-in live AWS contract tests for Lake Formation semantics that local
  emulators cannot faithfully prove:
  - table LF-Tag values with column-level overrides for the same LF-Tag key;
  - rejection of a single LF-Tag table-policy grant that combines `SELECT` with
    mutating permissions such as `INSERT`;
  - parsing of real Lake Formation LF-Tag response shapes.
- Adds `docs/testing.md` to explain default, emulator, and live AWS test layers.
- Adds a dedicated GitHub Actions job for the Moto emulator suite.

## Changed

- Adds `botocore` to the `test` extra so Stubber contract tests run in the
  default CI test job.
- Adds a `mock-aws` extra for installing Moto-backed local emulator tests.

## Upgrade

```bash
python -m pip install --upgrade lfguard
lfguard --version
```

For package development:

```bash
python -m pip install -e ".[test]"
python -m unittest discover -s tests
```
