# lfguard 0.6.0

`lfguard 0.6.0` adds a stable current-state provider boundary, cache-backed
live workflows, and first-class Lake Formation data cells filter definitions.

## Added

- `CurrentStateProvider` is now the integration boundary for current-state
  loading, with snapshot, file, lazy, and cached provider implementations.
- `CachedCurrentStateProvider` stores current state in a JSON cache envelope
  keyed by the desired-state scope fingerprint.
- `lfguard audit`, `plan`, `explain`, and desired-driven `apply` support
  `--current-cache`, `--refresh-current-cache`, and
  `--current-cache-max-age`.
- Desired and current state can model `data_cells_filters` definitions for Lake
  Formation `DataCellsFilter` bodies, including row filters, all-rows filters,
  included columns, and excluded columns.
- Plans can create data cells filters and can update or delete them when
  `--allow-data-cells-filter-updates` or
  `--allow-data-cells-filter-deletes` is supplied.
- Audit, lint, and explain reports now include data cells filter definition
  drift, duplicate identity checks, and matching filter-body evidence.

## Changed

- Live current-state loading now fetches data cells filter definitions named in
  desired state or referenced by data cells filter grants.
- `lfguard import` can scaffold `data-cells-filters`; import remains bounded to
  tables discovered from Lake Formation grants.
- Generated schemas and state-format docs include the `data_cells_filters`
  top-level section.

## Tests

- Adds unit coverage for provider cache hit, refresh, scope mismatch, expiry, and
  CLI cache behavior.
- Adds model, planner, audit, lint, explain, and AWS Stubber coverage for data
  cells filter definitions.
- Adds Stubber coverage for `CreateDataCellsFilter`, `UpdateDataCellsFilter`,
  `DeleteDataCellsFilter`, `GetDataCellsFilter`, and `ListDataCellsFilter`
  request shapes.

## Upgrade

```bash
python -m pip install --upgrade lfguard
lfguard --version
```
