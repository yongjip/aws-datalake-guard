# lfguard 0.6.2

`lfguard 0.6.2` hardens current-state cache usage for direct library
integrations and shared automation environments.

## Fixes

- Direct library users can now create AWS-scoped cache providers with
  `CachedCurrentStateProvider.for_aws(...)`, which records the same provider,
  profile, region, and catalog context that the CLI uses.
- `aws_current_state_provider_context(...)` is exported for integrations that
  need to build or inspect AWS cache context explicitly.
- The CLI now reuses the shared AWS cache-context helper instead of carrying its
  own equivalent implementation.
- Cache writes now use unique temporary files in the target directory before
  atomic replace, avoiding collisions when parallel jobs share a cache
  directory.

## Notes

- Existing generic `CachedCurrentStateProvider(..., provider_context={})`
  behavior remains available for provider-independent snapshots and tests.
  Live AWS library integrations should prefer `for_aws(...)`.
