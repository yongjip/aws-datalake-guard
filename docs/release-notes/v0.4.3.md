# lfguard 0.4.3

`lfguard 0.4.3` closes remaining catalog-scoped named LF-Tag expression gaps
and consolidates shared state indexing.

## Fixed

- State serialization now preserves same-name LF-Tag expressions in different
  catalogs. Duplicate expression names are emitted in list form so round trips
  no longer drop all but one catalog.
- Lint now validates named LF-Tag expression references by `(catalog_id, name)`.
  Unscoped grants match an unscoped expression first, or a single scoped
  expression only when that match is unambiguous.

## Changed

- Plan, audit, lint, explain, and AWS current-state loading now share internal
  catalog-aware state indexing helpers.
- `lfguard summary --output json` includes additive `lf_tag_expression_ids`
  values so same-name expressions in different catalogs remain visible. The
  existing `lf_tag_expression_names` field is retained.

## Tests

- Adds regression coverage for duplicate named LF-Tag expression serialization,
  catalog-scoped lint references, ambiguous unscoped named-expression grants,
  and catalog-aware summary output.

## Upgrade

```bash
python -m pip install --upgrade lfguard
lfguard --version
```
